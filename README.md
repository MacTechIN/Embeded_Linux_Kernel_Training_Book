# 임베디드 리눅스 커널 트레이닝 북
# Embedded Linux Kernel Training Book

> **초보 엔지니어를 위한 임베디드 리눅스 시스템 개발 실습 교과서**
> A step-by-step hands-on tutorial for beginner engineers learning Embedded Linux.

---

## 프로젝트 소개

이 저장소는 **임베디드 리눅스를 처음 배우는 엔지니어**가 아무것도 모르는 상태에서 시작하여, 자신의 손으로 리눅스 커널을 빌드하고, 타겟 보드에 올리고, 드라이버를 작성하는 수준까지 도달할 수 있도록 설계된 **단계별 실습 교과서**입니다.

모든 매뉴얼은 **마이크로 프로세스 단위**로 구성되어 있어, 처음 보는 명령어 한 줄, 옵션 하나의 의미까지 빠짐없이 설명합니다. 개념 설명 → 실습 → 결과 확인이 반복되는 구성으로 따라가기 쉽게 만들어졌습니다.

## 저장소 구조

```
.
├── README.md               ← 지금 보고 있는 파일
├── CLAUDE.md               ← Claude Code 작업 지침
├── docs/                   ← 공식 학습 산출물
│   ├── manual001.md        ← M001: 임베디드 리눅스 첫걸음
│   ├── manual002.md        ← M002: 호스트 개발 환경 구축
│   └── km.md               ← 시계열 학습 메모 (KM 로그)
└── doc/                    ← 작업용 내부 문서
    ├── plan.md             ← 커리큘럼 계획 및 진행 상태
    └── research.md         ← 매뉴얼 작성 전 리서치 메모
```

- `docs/` — 학습자가 읽는 **완성된 매뉴얼**과 KM 로그.
- `doc/` — 매뉴얼 집필에 사용하는 **내부 작업 문서** (연구·계획).

## 매뉴얼 구성 원칙

각 매뉴얼은 다음 구조를 따릅니다.

1. **학습 목표** — 이 단계를 마치면 무엇을 할 수 있는가
2. **사전 지식 점검** — 필요한 것 / 없어도 되는 것
3. **개념 설명** — 용어 최초 등장 시 그 자리에서 풀이
4. **단계별 실습** — 명령 → 예상 출력 → 의미 설명 → 다음 단계
5. **소스 코드** — 복붙 가능한 완전한 형태 (생략·축약 없음)
6. **검증** — 예상 결과 및 확인 방법
7. **트러블슈팅 / FAQ**
8. **다음 단계 안내**
9. **부록** — 명령어 한 줄 풀이 표
