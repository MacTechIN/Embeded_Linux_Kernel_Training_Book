# 개념 03: 핵심 용어 7가지

이 7개 용어는 임베디드 리눅스 개발에서 계속 나옵니다. 각 용어의 의미를 자신의 말로 설명할 수 있어야 합니다.

## 1. Host (호스트)

**호스트** = 개발자가 개발하는 데 사용하는 PC

```text
┌─────────────────┐
│  Host (호스트)   │
│                 │
│  내 개발 PC      │
│  - 주로 x86-64  │
│  - Ubuntu       │
│  - 코드 작성    │
│  - 컴파일       │
└─────────────────┘
```

### 예시
- "호스트에서 코드를 작성한다"
- "호스트에서 크로스 컴파일을 수행한다"
- "호스트는 x86_64 CPU를 사용한다"

---

## 2. Target (타겟)

**타겟** = 우리가 개발한 프로그램이 실제로 실행될 기기 (임베디드 보드)

```text
┌──────────────────┐
│ Target (타겟)     │
│                  │
│ 실행 환경        │
│ - Raspberry Pi   │
│ - BeagleBone     │
│ - 산업용 보드    │
│ - ARM CPU        │
└──────────────────┘
```

### 예시
- "타겟 보드는 ARM Cortex-A7 CPU를 사용한다"
- "타겟에서 프로그램이 실행된다"
- "타겟의 메모리는 512 MB이다"

### Host vs Target

| 항목 | Host | Target |
|------|------|--------|
| 역할 | 개발 환경 | 실행 환경 |
| CPU | x86-64 또는 ARM64 | 보통 ARM |
| 메모리 | 8 GB 이상 | 수백 MB ~ 수 GB |
| 목적 | 개발 작업 | 특정 기능 수행 |

---

## 3. Cross-Compile (크로스 컴파일)

**크로스 컴파일** = 호스트 CPU용이 아닌 **다른 아키텍처**의 바이너리를 생성하는 컴파일

```text
호스트 (x86-64 CPU)에서 컴파일
        ↓
      gcc (x86-64용) ─→ x86-64 바이너리 (네이티브 컴파일)
arm-linux-gnueabihf-gcc ─→ ARM 바이너리 (크로스 컴파일)
```

### 왜 필요한가?

호스트와 타겟의 CPU 아키텍처가 다르면 같은 소스 코드도 다른 기계어로 컴파일됩니다.

### 예시
- "크로스 컴파일러를 사용해 ARM 바이너리를 만든다"
- "호스트의 gcc는 네이티브 컴파일, arm-linux-gnueabihf-gcc는 크로스 컴파일을 수행한다"

---

## 4. Toolchain (툴체인)

**툴체인** = 임베디드 개발을 위해 필요한 **여러 도구들의 모음**

### 주요 도구들

| 도구 | 역할 |
|------|------|
| `gcc` | C/C++ 컴파일러 |
| `as` | 어셈블러 (ASM → 기계어) |
| `ld` | 링커 (오브젝트 파일 → 실행 파일) |
| `nm` | 심볼 테이블 분석 |
| `objdump` | 바이너리 역어셈블 |
| `strip` | 불필요한 심볼 제거 |
| `gdb` | 디버거 |

### 예시
- "ARM 크로스 툴체인을 설치한다"
- "gcc, ld, as가 모두 ARM용이어야 한다"

---

## 5. Bootloader (부트로더)

**부트로더** = 타겟 기기가 **전원을 켰을 때 가장 먼저 실행되는 프로그램**

### 역할

```text
┌──────────┐
│ 전원 ON  │
└────┬─────┘
     ↓
┌──────────────────────┐
│   Bootloader         │ ← 가장 먼저 실행
│ - CPU 초기화         │
│ - 메모리 초기화      │
│ - 커널 로드          │
└────┬─────────────────┘
     ↓
┌──────────────┐
│ Linux Kernel │
└────┬─────────┘
     ↓
┌──────────────┐
│ 애플리케이션 │
└──────────────┘
```

### 유명한 부트로더
- U-Boot (가장 일반적)
- GRUB (PC 리눅스)
- Das U-Boot

### 예시
- "부트로더가 커널을 메모리에 로드한다"
- "부트로더는 커널보다 먼저 실행된다"

---

## 6. Kernel (커널)

**커널** = 운영체제의 핵심, **하드웨어와 애플리케이션 사이의 중개자**

### 주요 역할

| 역할 | 설명 |
|------|------|
| CPU 관리 | 프로세스 스케줄링 |
| 메모리 관리 | 가상 메모리, 메모리 보호 |
| 장치 제어 | 디바이스 드라이버 |
| 파일 시스템 | 저장 공간 관리 |
| 네트워킹 | 통신 스택 (TCP/IP) |

### 구조

```text
┌──────────────────────────┐
│   애플리케이션 프로그램    │
├──────────────────────────┤
│   (System Call 인터페이스) │
├──────────────────────────┤
│   Linux Kernel           │
│ - 프로세스 관리          │
│ - 메모리 관리            │
│ - 파일 시스템            │
│ - 네트워크               │
├──────────────────────────┤
│   하드웨어 드라이버       │
├──────────────────────────┤
│   CPU, 메모리, 디스크    │
└──────────────────────────┘
```

### 예시
- "커널 버전은 5.10.0이다"
- "커널이 프로세스를 관리한다"

---

## 7. Rootfs (루트파일시스템)

**루트파일시스템** = 타겟 기기의 파일 시스템, 커널이 필요한 모든 파일들의 집합

### 포함되는 것들

```text
rootfs/
├── bin/          필수 실행 파일 (ls, cp, etc.)
├── sbin/         시스템 실행 파일 (mount, ifconfig)
├── lib/          공유 라이브러리 (libc.so.6 등)
├── etc/          설정 파일 (hostname, network config)
├── root/         루트 사용자 홈 디렉토리
├── home/         일반 사용자 홈 디렉토리
├── var/          로그, 임시 파일
├── tmp/          임시 파일
├── proc/         프로세스 정보 (커널이 제공)
├── sys/          시스템 정보 (커널이 제공)
└── dev/          디바이스 파일 (하드웨어 접근)
```

### 생성 도구
- Buildroot (가장 인기)
- Yocto/BitBake (산업용)
- Busybox (최소 버전)

### 예시
- "rootfs를 생성해야 타겟이 부팅된다"
- "rootfs 크기는 32 MB이다"

## 흐름도: 7가지 용어가 어떻게 연결되는가

```text
┌─────────────────────────────────────────────────────────┐
│  Host (호스트)                                            │
│  개발 PC에서 Toolchain 사용                              │
│                                                          │
│  소스 코드 ─→ 크로스 컴파일 ─→ ARM 바이너리             │
│       ↑                              │                   │
│       └──────────────────────────────┘                   │
│                                                          │
│  (Cross-Compile + Toolchain 사용)                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
                              ↓
            ┌────────────────────────────────┐
            │  Target (타겟 기기)              │
            │  Raspberry Pi / 임베디드 보드    │
            │                                │
            │  [Bootloader]  ← 전원 ON       │
            │       ↓                        │
            │  [Kernel]     ← 부팅           │
            │       ↓                        │
            │  [Rootfs]     ← 파일 시스템    │
            │       ↓                        │
            │  [애플리케이션] ← 실행          │
            │                                │
            └────────────────────────────────┘
```
