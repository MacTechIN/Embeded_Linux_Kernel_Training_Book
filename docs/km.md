# Embedded Linux 지식 관리 (KM)

## [2026-05-04 02:10] 임베디드 리눅스 개발 기초 자료 (v0.1.0)

### 1. 임베디드 리눅스 시스템 개발을 왜 해야 하는가?
임베디드 시스템은 범용 컴퓨터와 달리 특정 목적을 수행하기 위해 설계된 시스템이다. 리눅스를 임베디드 환경에서 개발해야 하는 이유는 다음과 같다.
- **최적화된 성능**: 제한된 하드웨어 자원(CPU, Memory) 내에서 최대의 성능을 끌어내기 위함이다.
- **안정성 확보**: 불필요한 기능을 제거하여 시스템 장애 요소를 최소화하고, 장시간 안정적인 운영을 보장하기 위함이다.
- **비용 절감**: 하드웨어 사양을 최소화하면서도 필요한 기능을 완벽히 구현함으로써 양산 비용을 낮출 수 있다.

### 2. 무엇이 유리한가? (임베디드 리눅스의 장점)
- **오픈 소스 에코시스템**: 방대한 라이브러리와 커뮤니티 지원을 통해 개발 기간을 단축할 수 있다.
- **유연한 커스터마이징**: 소스 코드가 공개되어 있어 커널 레벨부터 애플리케이션 레벨까지 시스템의 모든 부분을 제어할 수 있다.
- **강력한 네트워킹**: 다양한 네트워크 프로토콜 스택을 기본적으로 지원하며, 보안 업데이트가 빠르다.

### 3. 왜 그냥 리눅스가 아닌 '하드웨어 맞춤형' 임베디드 리눅스를 사용하는가?
범용 리눅스 배포판(Ubuntu, Debian 등)을 그대로 사용하지 않고 하드웨어에 맞게 Tailoring(재구성)하는 이유는 다음과 같다.
- **부팅 속도 최적화**: 범용 리눅스는 수많은 드라이버를 로드하느라 부팅이 느리지만, 맞춤형 리눅스는 필수 장치만 초기화하여 즉각적인 가동이 가능하다.
- **이미지 경량화**: 불필요한 패키지와 서비스를 제거하여 수십 MB 수준의 작은 이미지로 플래시 메모리에 탑재할 수 있다.
- **실시간성(Determinism)**: 하드웨어 인터럽트 처리나 스케줄링 정책을 특정 목적에 맞게 튜닝하여 예측 가능한 응답 시간을 확보할 수 있다.
- **보안성 강화**: 사용하지 않는 포트나 서비스를 제거하여 공격 지점을 원천적으로 차단한다.
