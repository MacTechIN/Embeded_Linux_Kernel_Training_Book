# Implementation Plan - Embedded Linux Knowledge Management (km.md)

## 1. 개요
이 계획은 `./docs/km.md` 파일에 임베디드 리눅스 시스템 개발의 기초 자료를 기록하기 위한 것입니다. 사용자 규칙에 따라 시계열로 기록하며, 기존 내용을 변경하지 않고 추가(Append)하는 방식으로 진행합니다.

## 2. 접근 방식
- **문서 구조**: 시계열 기록을 위해 각 항목에 타임스탬프 또는 순번을 부여합니다.
- **내용 구성**: 
    1. 임베디드 리눅스 시스템 개발의 필요성
    2. 일반 리눅스 대비 유리한 점
    3. 하드웨어 맞춤형(Tailoring) 사용의 이유
- **원문 유지**: 사용자의 요청대로 원문의 의도를 충실히 반영하여 기록합니다.

## 3. 상세 구현 단계
### 단계 1: 기초 자료 기록 (v0.1.0) - [완료]
- `docs/km.md` 파일을 생성하고 첫 번째 기록을 작성합니다.
- 작성 일시: 2026-05-04 (현재 시간 기준)
- 주요 내용:
    - 하드웨어 제약 조건 하에서의 성능 최적화 필요성
    - 오픈소스 기반의 비용 및 유연성 장점
    - 커널 경량화 및 부팅 속도 최적화의 필요성

## 4. 파일 경로
- `docs/km.md`: 지식 관리 메인 문서
- `doc/research.md`: 리서치 내용 백업
- `doc/plan.md`: 현재 계획 문서

## 5. 트레이드오프
- **범용성 vs 최적화**: 범용 리눅스는 사용이 간편하지만 리소스 낭비가 심함. 임베디드 리눅스는 개발 공수가 들어가지만 하드웨어 효율을 극대화할 수 있음.
- **Append 방식**: 문서가 길어질 수 있으나 히스토리 추적에 용이함.

## 6. 기타 원칙
- **인풋 페이징**: 향후 데이터 조회 기능 구현 시 오프셋 페이징 대신 인풋 페이징(Cursor-based) 방식을 고려함.
- **명칭**: 'Embedded Linux', 'Tailoring', 'Kernel Optimization' 등의 용어를 명확히 사용함.

## [2026-05-04 02:25] 실습 교과서 시리즈 시작 — 매뉴얼 체계 수립 및 M001 작성 (v0.3.0)

### 1. 배경 및 목적 변경
이 저장소의 목적이 **단순 KM 로그**에서 **초보 엔지니어용 임베디드 리눅스 실습 교과서 제작**으로 명확해졌다. 이에 따라 산출물 형식과 워크플로를 다음과 같이 정비한다.

### 2. 산출물 형식 (이후 모든 학습 단계에 적용)

- 학습 단계마다 `docs/manualNNN.md` 1개 파일 생성 (NNN: 0-padded 3자리, 학습 순서).
- `docs/km.md`는 시계열 학습 메모로 계속 운영(append-only).
- 코드 예제 디렉토리(`examples/manualNNN/`)는 필요해지는 시점에 사용자와 합의 후 추가.

### 3. 매뉴얼 작성 원칙 (모든 매뉴얼 공통)

- **마이크로 프로세스화**: 한 매뉴얼 안에서도 단계를 잘게 쪼개고 명령어/옵션/출력의 의미를 모두 설명한다.
- **실습·코드·설명 인터리브**: 개념 → 작은 실습 → 결과 확인 → 다음 개념.
- **초보 가정**: 처음 등장한 용어는 그 자리에서 풀이.
- **복붙 가능한 완전 명령**: 생략(`...`) 금지.
